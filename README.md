# PostgreSQL
## Технологическая практика, 5 семестр, кафедра системного программирования

### Программное обеспечение
Для выполнения заданий практикума использовалось программное обеспечение СУБД с 15.

[Документация СУБД PostgreSQL Pro](https://postgrespro.ru/docs/postgrespro/15/index), являющейся расширением СУБД PostgreSQL. Также есть англоязычная [документация СУБД PostgreSQL 15](https://www.postgresql.org/docs/15/index.html).

### Описание лабораторных
Первая лабораторная работа является примером из документации PostgreSQL и несёт ознакомительный характер, последующие лабораторные выполнены для предметной области "Университет". Выполнение лабораторных работ нацелено на практическое применение навыков, полученных после изучения документации СУБД PostgreSQL 15. Каждая следующая работа основывается на результатах предыдущей и логически их нее следует.

_Лабораторная 1_

Написаны 4 SELECT-запроса к модельной базе данных, решающие полезную задачу из предметной области.

_Лабораторные 2-5_

- Спроектирована схема база данных для изучаемой предметной области, схема базы представлена в виде ER-диаграммы, содержащей таблицы и связи между ними.
- Написан SQL-скрипт для создания спроектированной БД и заполнения её небольшим количеством данных (10 экземпляров для каждой стержневой сущности и 20 для ассоциативных).
- Подготовлены выборки для БД, которые имеют осмысленное значение для предметной области, для них составлены SQL-скрипты. Сформулированы запросы на изменение и удаление из БД, среди них есть те, которые вызывают срабатывание ограничений целостности.
- Подготовлен скрипт для создания триггера, а также два запроса для проверки и демонстрации его работы. В файле .pdf приведены примеры для проверки наличия аномалий (потерянное изменение, грязное чтение, неповторяющееся чтение, фантомы) при параллельном исполнении транзакций на различных уровнях изолированности.

_Лабораторные 6-9_

- Спроектирована схема базы данных для аналитики в предположении, что приложение, для которого была сделана БД в лабораторной 2, стало очень популярным. Создана денормализованная структура данных, подходящая для OLAP-анализа, использована концепция "звезды". Написан SQL-скрипт для создания спроектированной БД и заполнения её данными (таблица фактов содержит 100 млн строк, таблицы измерений по 1 млн строк). Для сокращения скорости заполения БД данные генерируются отдельно в файлы .csv с использованием python-скриптов.
- Создан пользователь test и ему выдан доступ к БД. Составлены скрипты присвоения пользователю прав доступа к таблицам. Составлены SQL-скрипты для создания нескольких представлений для упрощения манипулирования данными. От имени нового пользователя выполнены выборки из таблиц и представлений, операторы с ограниченными правами доступа.
- Составлен SQL-скрипт для создания нескольких функций. В них продемонстрированы циклы, ветвления, переменные, курсоры и исключения.
- Созданы индексы, ускоряющие выполнение запросов к таблицам, их работа продемонстрирована на нескольких примерах. Написан SQL-скрипт для выполнения секционирования таблицы фактов.
